# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

mcshell is a Wayland desktop shell (status bar, notifications, app launcher, quick settings) built entirely in QML using [mcs-qs](https://github.com/Stoica-Mihai/mcs-qs) — our in-house fork of [Quickshell](https://quickshell.outfoxxed.me/), a Qt6/QML-based shell toolkit for Wayland compositors. It targets the **niri** compositor specifically via the native `Quickshell.Niri` IPC module. The fork carries a small IPC ergonomics patch (optional positional args — stock Quickshell rejects any arg-count mismatch) plus the native modules this shell relies on (`Quickshell.Networking`, `Quickshell.Services.Polkit`, `Quickshell.Wayland._DataControl`, `Quickshell.Wayland._GammaControl`, `VibrantColor`, etc.).

## Running

```sh
make start       # start shell in background
make stop        # stop running shell
make restart     # stop + start
make test        # smoke test: start shell, run all IPC commands, check for errors/warnings
```

Or directly: `./start.sh` / `mcs-qs -c mcshell`

The shell requires a running Wayland session with wlr-layer-shell support (niri, Hyprland, Sway, etc.). There is no build step for mcshell itself — mcs-qs interprets QML directly.

**QML edits hot-reload — skip `make restart` after editing a `.qml` file.** Quickshell watches the config tree and re-parses on save, so changes are live within a frame. Restarting wastes seconds re-loading and clears any open popups/panels the user is interacting with. Restart is appropriate when switching binaries, changing env vars, or recovering from a parse error that left the tree broken.

**When you do restart, don't sleep-pad after it.** `make restart` does `stop && sleep 1 && start &`, so the command returns in ~1.5s and the new shell is IPC-ready ~1s later. Chain at most one `sleep 2` before the next `mcs-qs ipc call`. Padding a `sleep 6/8` "to be safe" multiplied across iterations is what wastes minutes per session — past sessions burned real time on this.

## Architecture

**Entry point:** `shell.qml` — creates a `ShellRoot` containing a `StatusBar` per screen, plus `NotificationPopup`, `AppLauncher`, `KeybindPanel`, `LockScreen`, `WallpaperRenderer`, `ScreenshotOverlay`, `Recording`, and `PolkitDialog`, plus an `IdleMonitor` that drives auto-lock. Hosts the central IPC handler (`_dispatchPanel`, `_dispatchLauncher`, `_resolveMode`, `_panelModes` registry).

**Module layout** — each directory is a QML module imported as `qs.<DirName>`:

| Module | Purpose |
|---|---|
| `Config/` | `Theme` singleton — colors (8 palettes + auto wallpaper theming via `VibrantColor`), layout constants, typography, icon codepoints. `UserSettings` singleton — persistent user preferences (`~/.config/mcshell/settings.json`) via `FileView` + `JsonAdapter` with `watchChanges` for live-reload. |
| `Core/` | Non-visual shared components — `SafeProcess` (process wrapper with error logging), `ShellActions` singleton (session actions: lock, logout, reboot, shutdown, wallpaper — callable from any component), `Brightness` singleton (screen brightness via `brightnessctl`, event-driven via `FileView` sysfs watcher), `HolidayService` singleton (fetches public holidays from Nager.Date keyed by `UserSettings.weatherCountryCode`, caches per year, lazy-fetches on first binding access, clears cache on country change), `NotificationDispatcher` singleton (internal notification send helper with image support), `WeatherCodes` singleton (WMO weather code → icon/description mapping), `JsonFetcher` (reusable HTTP JSON fetch with error handling), `ConnectivityRetry` (auto-retry on network reconnect), `OverlayWindow` (fullscreen layer-shell overlay base). |
| `Bar/` | `StatusBar` — three parallelogram segments (left/center/right) with Shape backgrounds and static gradient border (`BarBorder` — solid or gradient style via `UserSettings.barBorderStyle`). Fixed width for left/right segments (`Theme.barSideWidth`). Left segment: launcher + workspaces locked left, active window right. Center segment: `Clock` (left-click opens calendar, right-click opens clock-settings), optional recording indicator, separator, `Weather` (left-click opens forecast, right-click opens weather in edit mode). Right segment: media locked left, systray + capsule (WiFi, Bluetooth, Volume, Battery, notification bell) locked right. Three `AnimatedPopup` instances — `leftDropdown` (keybinds), `centerDropdown` (calendar / weather / clock-settings), `sharedDropdown` (volume / notifications / media / sysinfo / sysinfo-settings / trayicons) — all xdg-popups anchored to their bar segment, swapping content via `activePanel`. `CalendarPopup` reads public holidays from `HolidayService` — holiday days get a dot marker and the viewed month's holidays are listed below the grid. Components: `Workspaces`, `ActiveWindow`, `Clock`, `CalendarPopup`, `Weather`/`WeatherPopup`, `ClockSettingsPopup`, `SysInfoSettingsPopup`, `BarSegment` (filled parallelogram + border), `Media` (MPRIS — bar controls only, popup content in shared dropdown), `Volume`/`VolumeSlider`/`VolumeWaveform`/`AppVolume`/`AppStreamProvider`, `Battery`, `SysTray` (collapsed ring indicator)/`SysTrayPanel` (expanded icon grid + context menu), `SysWaveform`/`WaveformBars`/`SysInfoPanel` (CPU/RAM/temp/disk/network/GPU dashboard, toggled via `sysInfoEnabled` setting; GPU rows: utilization/VRAM/power/clock for AMD via sysfs + NVIDIA via NVML), `CapsuleItem`, `MenuItem`, `Recording` (wf-recorder driver). |
| `Notifications/` | DBus notification daemon via `NotificationServer`. `NotificationPopup` manages a `ListModel` of active popups; `NotificationCard` renders each one with urgency-colored accent bar and action buttons. Auto-clean timer (event-driven, no polling) removes notifications older than the user-configured threshold (`notifAutoClean` setting). |
| `Launcher/` | `AppLauncher` — generic fullscreen carousel container with a three-level navigation state (`property string level: "view" \| "list" \| "edit"` plus derived `inView`/`inList`/`inEdit`/`inCarousel` helpers; `view` = search/browse categories, `list` = scroll cards, `edit` = drill into the selected card). Each tab is a `LauncherCategory` component with a `tabName` identifier (`CategoryApps`, `CategoryClipboard`, `CategoryWifi`, `CategoryBluetooth`, `CategoryWallpaper`, `CategorySettings`) and declares `property var supportedModes` (base is `["view", "list"]`; Settings adds `"edit"`). Categories can provide an optional `headerDelegate` for content above the carousel (used by `CategoryWallpaper` for the monitor strip). Tabs are opened by name via `openTab(name, mode, target)` — `mode` is a level, `target` is an optional item id to pre-select (categories override `onOpenTarget(target)` to interpret; Settings uses it for card ids). All categories use `setItems()` for lazy-loaded progressive model growth. Shared components: `CarouselStrip` (card layout with `focused` border), `LauncherCategory` (base interface for all tabs with optional `headerDelegate`), `DisabledCard` (off/scanning states), `SettingsCard` (data-driven settings sub-routing), `SettingsPanel` (base for settings sub-panels with keyboard navigation), `SettingsRow` (highlighted row for settings panels), `SelectionRow` (checkmark + label row for device/option selection), `SettingsProgressBar` (read-only progress bar for settings), `StatusHintText` (connection status display), `StatusTracker` (async operation state tracker). |
| `LockScreen/` | Wayland session lock (`ext_session_lock_v1`) with PAM authentication. |
| `Polkit/` | `PolkitDialog` — polkit agent that prompts for authentication when a privileged operation is requested. |
| `Wallpaper/` | `WallpaperRenderer` (background per screen with crossfade transitions, per-screen wallpapers via `perScreenMap`, configurable fill mode), `WallpaperScanner` (filesystem scanner for wallpaper folder), `WallpaperRotator` (auto-rotate at configurable interval with per-screen targeting and shuffle/sequential order). Wallpaper picker is in `Launcher/CategoryWallpaper` (monitor strip header, Tab to cycle target screen, screen badges on cards). |
| `KeybindHints/` | `KeybindParser` (KDL config parser with live file watching) + `KeybindPanel` (searchable, keyboard-navigable overlay UI). |
| `NotificationHistory/` | Notification history dropdown in the bar. Uses `Flickable` + `Column` + `Repeater` (not ListView) for accurate content-driven height sizing, capped at 3 visible entries with scrollbar for overflow. |
| `Screenshot/` | `ScreenshotOverlay` is a thin dispatcher that owns one `ScreenshotScreenOverlay` per screen via `Variants`; it detects the niri-focused output and broadcasts a capture request that only the targeted overlay acts on. Each per-screen overlay is an always-visible `OverlayWindow` with native Wayland screencopy + `mask: Region {}` + `opacity: 0` for idle state. The dispatcher pattern exists because reassigning a layer-shell surface's `screen` at runtime raced with Qt's Wayland `handleScreensChanged` and segfaulted on Qt 6.11. |
| `Widgets/` | Shared UI components — `AnimatedPopup` (panel API: `activePanel`, `anchorSection`, `togglePanel`/`openPanel`/`closePanel`, `autoPosition`), `AnimatedBorder` (pluggable card border animations: midpoint, clockwise, corners, fade), `IconButton`, `TextButton` (labeled button with hover highlight), `SliderTrack`, `ControlSlider`, `CyclePicker` (supports inline `◀ value ▶` text or pill mode wrapping the value in a `SkewPill` between `SkewArrow`s), `ParallelogramCard` (overridable `_skew`), `SkewRect`/`SkewArrow`/`SkewPill`/`SkewToggle`/`SkewCheck` (skewed-morphism primitives — parallelogram fill/stroke, triangular arrows matching pill diagonals, value-pinned label pills), `StyledTextField` (search fields), `MediaControls` (MPRIS transport buttons), `ActiveUnderline` (accent indicator), `HoverText`, `InfiniteText` (marquee scroll for long titles), `OptImage` (Image with async+PreserveAspectCrop defaults), `Separator` (1px divider with optional inset margins + overridable color), `ShakeAnimation` (horizontal shake for auth failure), `SmoothWheelHandler`, `ThemedScrollBar`, `ThemedTooltip`, `TriToggle`. |

**Key patterns:**
- **Native APIs first:** Workspaces and active window use `Quickshell.Niri` (reactive, event-driven via niri IPC socket). Network status uses `Quickshell.Networking`. Audio uses `Quickshell.Services.Pipewire`. Media uses `Quickshell.Services.Mpris`. System tray uses `Quickshell.Services.SystemTray`. Bluetooth uses `Quickshell.Bluetooth`. Battery uses `Quickshell.Services.UPower`. System resource monitoring (CPU / RAM / temps / net / disk / GPU) uses `Quickshell.Services.SysInfo` — shipped natively in mcs-qs v1.1.0+, polls /proc, /sys, hwmon, and dlopens libnvidia-ml.so.1 for NVIDIA GPU metrics. Screenshots use `Quickshell.Wayland.ScreencopyView` (native Wayland screencopy). Clipboard history uses `Quickshell.Wayland._DataControl` (native `ext-data-control-v1`). Night light uses `Quickshell.Wayland._GammaControl` (native `zwlr-gamma-control-v1`). Surface blur uses `Quickshell.Wayland.BackgroundEffect` (`ext-background-effect-v1` — needs niri ≥ 26.04). Auto-theming uses `VibrantColor` (CIELAB chroma-weighted hue extraction from wallpaper). CLI tools (`brightnessctl`, `nmcli`, `wf-recorder`) are only used where no native API exists.
- **Surface blur:** when `UserSettings.blurEnabled` is on, every visible surface attaches `BackgroundEffect.blurRegion: Region { ... }` and drops its background alpha (0.85/0.88 → 0.55) so the blur shows through. The bar uses the new mcs-qs `Region.polygon` property to follow the parallelogram segments exactly; rectangular surfaces (bar dropdowns, notifications, polkit dialog) use `Region { item: ... }`. Per-surface gates are required where a layer-shell surface stays alive across hide/show cycles (notifications gate on `notifModel.count > 0`; polkit gates on `agent.isActive`; launcher gates on `launcher.isOpen`) — otherwise the blur paints a phantom rectangle even when the visible content has zero opacity. The launcher's layer-shell surface is fullscreen so the wallpaper-blur extends edge-to-edge; its backdrop drops dim to `transparent` when blur is on so the wallpaper-blur carries the look. The bar lives on a higher layer and stays visible above it. niri's `ext-background-effect-v1` runs in xray mode by default and only samples the wallpaper layer for the blur source — app windows under a blurred surface are hidden, not blurred-but-visible. xdg-popups (the bar dropdowns) sample through their parent layer surface, so they show app-window blur instead. The toggle lives in Settings → Theme as the first row.
- **IPC mode dispatch:** Every panel-opening IPC in `shell.qml` takes an optional `mode: string` argument (`view` / `list` / `edit`). Bar panels are validated against a central `_panelModes` registry on `shell.qml`; launcher tabs are validated against each category's `supportedModes` property via `appLauncher.supportedModesFor(tabName)`. Both paths share a single `_resolveMode(kind, name, modes, mode)` helper that warns with the `mcshell IPC:` prefix and returns `null` for unknown panels or unsupported modes (the caller then skips the dispatch). `launcherX` IPCs additionally accept a second `target: string` argument that's passed to `activeCategory.onOpenTarget(target)` after the level is applied — categories override `onOpenTarget` to interpret the string (e.g. Settings treats it as a card id). `test.sh` has an `expect_warn` helper that asserts specific validation warnings fire, and the WARN counter filters the `mcshell IPC:` prefix out of pass/fail totals.
- **Error handling:** Fire-and-forget subprocess calls (brightness, wifi/bt CLI, power actions, wallpaper scan) use `SafeProcess` from `Core/` — it logs failures via `failMessage` and emits `failed()` / `finished()` signals. Calls that need to capture stdout (curl to JSON APIs in `Weather.qml`, `WeatherPopup.qml`, `HolidayService.qml`; `wf-recorder` in `Recording.qml`; `niri msg` in `ScreenshotOverlay.qml` / `shell.qml`) use raw `Process` with `StdioCollector` — they own their own error path via `onExited` / `onStreamFinished`.
- **Layer shell windows:** Top-level layer surfaces use `WlrLayershell` with namespaces prefixed by `mcshell` — `mcshell` (bar main surface), `mcshell-zone` (exclusion reservation), `mcshell-notifications`, `mcshell-launcher`, `mcshell-screenshot`, `mcshell-wallpaper`, `mcshell-polkit`. Bar dropdowns are xdg-popups (`AnimatedPopup`/`PopupWindow`) anchored to the bar's main surface — they don't get their own layer-shell namespace.
- **Popup dismissal:** The `StatusBar` uses a single fullscreen `PanelWindow` (`mainSurface`) with input masking (`mask: Region {}`) — bar-only input when no popup is open, fullscreen when a popup is open. A `MouseArea` behind all content catches outside clicks to dismiss. No separate clickCatcher window.
- **Icons:** Uses "Symbols Nerd Font" for all icons (Unicode codepoints, not icon names).
- **No build system:** Pure QML — no C++, no CMake, no npm. Files are loaded by mcs-qs at runtime.
- **Pixmap cache pinning for large thumbnail carousels:** Categories that render many image thumbnails (`CategoryWallpaper` with hundreds of files) instantiate a hidden `Repeater` of `OptImage`s keyed on every path to keep one reference alive per pixmap — without this, Qt evicts entries under cache pressure and re-entering the tab triggers cold decodes. Two hard requirements: (1) the cache-keeper MUST use the same `Image` subclass + identical `sourceSize` as the carousel delegates — a plain `Image` preloading the same URL produces a distinct cache entry that the carousel's `OptImage` won't find; (2) the expanded (full-res) image's `source` must be gated on `isCurrent` so only the selected card holds a 480p copy, otherwise all N delegates decode in parallel and blow past Qt's ~128 MB cache limit.
- **Parallelogram skew alignment:** The bar's search-bar canvas (`barDiagSlant`) and `ParallelogramCard` (`cardSkew`) are both parallelograms, but their skews have independent magnitudes and directions. When stacking one above another, centring their Item bounding rects leaves the adjacent edges offset by a visible Z-shape — `AppLauncher.calcRowX` applies a `skewPx - barDiagSlant/2` compensation so the card's top-edge visual centre lines up with the search bar's bottom-edge visual centre.

## System Dependencies

- **[mcs-qs](https://github.com/Stoica-Mihai/mcs-qs)** — our Quickshell fork (`mcs-qs` binary). Ships all native modules mcshell needs (no separate plugin install required).
- **niri** compositor
- **PipeWire + WirePlumber** — audio (native API)
- **NetworkManager** — network status (native API) + `nmcli` for WiFi connect with password
- **brightnessctl** — screen brightness
- **Fonts:** JetBrains Mono, Symbols Nerd Font

## Conventions

- **Native APIs first, CLI fallback only.** Always use QuickShell's built-in service APIs (`Quickshell.Niri`, `Quickshell.Services.Pipewire`, `Quickshell.Services.Mpris`, `Quickshell.Services.SystemTray`, `Quickshell.Services.Notifications`, `Quickshell.Services.UPower`, `Quickshell.Bluetooth`, `Quickshell.Networking`, etc.) before resorting to CLI tools. Native APIs are reactive and instant; CLI polling is laggy. Only use `SafeProcess` + CLI tools when no native QuickShell API exists.
- **Documentation references.**
  - Qt6 / QML: `https://doc.qt.io/qt-6/` (start page) and `https://doc.qt.io/qt-6/qtquick-index.html` for QML specifics. Per-type pages live at `https://doc.qt.io/qt-6/qml-qtquick-<type>.html` (e.g. `qml-qtquick-image.html`).
  - Qt source (when docs are vague on behavior — cache keys, eviction, internal state machines): `https://code.qt.io/cgit/qt/qtdeclarative.git/tree/` — reading the C++ is often faster than guessing.
  - Quickshell types (layer shell, Niri IPC, Services.*, native Wayland modules): `https://quickshell.org/docs/`.
- All colors and layout constants are centralized in `Config/Theme.qml` — reference `Theme.bg`, `Theme.accent`, `Theme.overlay`, `Theme.backdrop`, etc. rather than hardcoding `Qt.rgba()` values.
- Subprocess calls that don't need stdout capture must use `SafeProcess` from `Core/` so failures are logged uniformly. Calls that need stdout via `StdioCollector` (curl/JSON fetch, `wf-recorder`, `niri msg -j`) use raw `Process` and must handle their own error paths via `onExited` / `onStreamFinished` — never raw `Process` that silently swallows failures.
- Animation durations use Theme constants: `Theme.animFast` (100ms, hover), `Theme.animNormal` (150ms, state changes), `Theme.animSmooth` (200ms, transitions), `Theme.animCarousel` (350ms, carousel). Never hardcode duration values.
- **Large list performance:** All categories use `LauncherCategory.setItems()` which progressively windows the model — initially ~15 items, growing as the user navigates via `growItems()`. Model updates are deferred by 350ms to preserve carousel animations. Never set `model` directly; always use `setItems()`.
- Popup windows (`PopupWindow`) anchor to their trigger widget via `anchor.item` / `anchor.rect`.
- **DRY — no exceptions.**
  - **Extract shared components BEFORE using them.** If a pattern will be used more than once, write the shared component first, then use it everywhere. Never copy-paste and "refactor later."
  - **New features must use existing patterns.** Before writing any new code, check how existing similar features work. If the launcher has 5 tabs that all use a Repeater + CarouselStrip inside slidingRow, the 6th tab must use the same pattern — not a separate Row with custom positioning.
  - **Make the system generic, not special-cased.** If adding a new feature requires `if (feature === X)` branches in shared code, the architecture is wrong. Instead, define an interface that all features implement, and have the shared code dispatch generically. Example: `LauncherCategory.qml` defines the interface; each tab implements it; `AppLauncher.qml` uses `activeCategory.model` instead of `if (activeTab === 0) filteredApps else if...`.
  - **Hardcoded indices are a code smell.** `categories[1].clipboardLoaded` breaks when tabs are reordered. Use properties on the active category instead. Tabs are referenced by `tabName` strings (`openTab("settings")`), not by index.
  - **Key examples:** `CarouselStrip.qml` is the single card implementation for all launcher tabs. `LauncherCategory.qml` is the base interface for all tab categories (includes `_validIndex()` bounds check helper). `DisabledCard.qml` is shared across WiFi-off, BT-off, and scanning states. `CapsuleItem.qml` is the shared icon+label pattern in the bar capsule. `SettingsRow.qml` is the shared row for all settings panels. `SettingsProgressBar.qml` is the shared read-only progress bar for settings sliders. `SafeProcess.qml` wraps all subprocess calls. `AnimatedPopup` provides the shared dropdown API (`togglePanel`/`openPanel`/`closePanel`) used by both center and right bar segments. `MediaControls.qml` is the shared MPRIS transport (prev/play/next) used in both the bar and the media dropdown. `StyledTextField.qml` is the shared flat search field. `SkewTextField.qml` is the shared parallelogram-skewed search field used by `WeatherEditView` and `KeybindPanel`. `ActiveUnderline.qml` is the shared accent indicator for active items. `WaveformBars.qml` is the shared 8-bar waveform skeleton used by both `VolumeWaveform` and `SysWaveform`. `TextButton.qml` is the shared text-label button used by `PolkitDialog`.
- **Disabled/muted state convention:** When a feature is disabled (volume muted, DND active), its icon must be red (`Theme.red`) with a slashed variant. Red takes priority over hover color — never let hover override the disabled state.
- **Keyboard-first design.** mcshell targets niri's keyboard-driven paradigm. Interactive features should be controllable via keybinds, not require mouse clicks. Buttons are a last resort — prefer keybind hints in the UI.
- **Verify before handing off.** Before presenting changes for testing, always run `mcs-qs -c mcshell` (or reload the shell) and verify that the code at minimum loads without errors. Check QML console output for warnings. If the shell doesn't load, fix it — don't hand broken code to the user. When refactoring, verify that existing functionality still works: tabs switch, cards render, keyboard shortcuts respond.
